## Features
This project simulates a complete relational database for a perfume store network, structured into six main technical phases:
1) Table creation: Designs a normalized schema with tables for Branduri, Categorii, Magazine, Parfumuri, ParfumuriCategorii, and Oferte
   
3) Structure and constraint Updates
   
5) Record addition: Populates the database with some of my favourite perfumes
   
7) Data update and recovery
   
9) Advanced queries (1-30): Includes 30 complex SELECT queries, utilizing:
    
* Table joins and subqueries
* Grouping functions with HAVING clauses
* String, date, and math functions
* Set operators
* Outer joins
  
6) Database objects: Implements practical database objects, including:
* custom views
* indexes
* sequences
* synonyms
